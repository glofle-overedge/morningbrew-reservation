# B-03-O3 · PRD 대조 점검 기록 (Day 03)

- **작성일:** 2026-07-12
- **시나리오:** 시나리오 1 — 카페 「모닝브루」
- **사업명:** 모닝브루 (사장 정해린) — 실습 예시
- **핵심 유저 스토리:** 출근길 직장인이 오늘 메뉴와 픽업 시간을 보고 예약한다
- **만든 화면:** 예약 입력 폼 + 예약 목록 + 삭제 + 빈 상태 (+ 유효성·피드백 강화)

## 산출물 체크

- **B-03-O1** ✅ 입력 → 추가 → 목록 표시 → 삭제가 모두 동작하고, 목록이 비면 빈 상태 안내가 보인다. 입력 필드(이름·연락처·픽업 시간·인원)는 모닝브루 예약 흐름에 맞게 구성.
- **B-03-O2** ✅ (선택) 빈 칸 제출이 막히고 각 칸 아래 "필수 항목입니다" 표시, '저장 중...' → 초록색 완료 안내(3초 후 사라짐), 삭제 시 확인 창 동작.
- **B-03-O3** ✅ 본 기록.

## PRD 핵심 흐름 구현 여부

- **구현함.** 어제 PRD lite의 핵심 흐름("손님이 예약을 넣으면 사장이 목록에서 확인한다")을 폼·목록·삭제·빈 상태로 그대로 구현.
- **빠진 단계:** 없음(핵심 유저 스토리 1개 범위 기준). 메뉴 선택·픽업 알림 등은 다음 유저 스토리로 이어 붙일 예정.

## 보완 결과

- 추가 요청으로 유효성 검사와 처리 중·완료·삭제 확인 피드백을 보완함(B-03-O2).

## 동작 확인 체크리스트

| 확인 항목 | 결과 |
|---|---|
| 빈 상태 (처음 열림) | ✅ 안내 문구 표시 |
| 정상 추가 | ✅ 목록에 추가됨 |
| 여러 개 추가 (2~3건) | ✅ 목록 안 깨짐 |
| 삭제 | ✅ 원하는 항목만 지워짐 |
| 모두 삭제 | ✅ 다시 빈 상태로 복귀 |
| 새로고침 | ✅ 데이터 사라짐 — 오늘은 정상 |
| 모바일 375px | ✅ 폼·목록 세로로 쌓임(720px 이하 1열 전환) |

## 공개·제출 전 점검

- ✅ 이름·연락처는 실습용 가짜 데이터로만 입력 (실제 손님 정보 금지)
- ✅ 다른 기기/시크릿 창에서도 폼·목록·빈 상태 동작
- ✅ 375px에서 폼과 목록 겹침 없음
