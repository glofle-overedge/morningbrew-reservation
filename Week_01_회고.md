## 가장 중요한 배움

- **배포·재배포 흐름:** GitHub 저장소에 커밋·푸시하고 같은 Vercel 프로젝트로 재배포하면, 새 프로젝트를 만들지 않고 같은 공개 URL이 새 버전으로 갱신된다는 것을 익혔다.
- **저장 경계:** localStorage는 브라우저·origin(주소)별로 분리 저장된다 — localhost와 배포 URL의 데이터가 서로 다른 것이 정상이라는 점.
- **통합의 핵심은 디자인이 아니라 "같은 사업 이야기":** 기능→메시지(PRD 대조)→디자인 순서로 합쳐야 어디서 어긋났는지 짚을 수 있다.

## 가장 어려웠던 부분

- 경로에 한글이 있어 Vercel CLI가 처음 배포에 실패 → 파일을 ASCII 경로로 복사해 배포하는 우회가 필요했다. (자동 재배포로 바꾸려면 Vercel 대시보드에서 GitHub 저장소를 연결하는 방법을 더 익혀야 함.)
- Day 04 저장 코드를 유지한 채 통합하는 것 — 통합 중 저장 흐름이 깨지지 않도록 STORAGE_KEY(`reservations`)를 그대로 유지했다.
